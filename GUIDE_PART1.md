# Часть 1: Архитектура и стек {#часть-1}

## Что мы строим

Интернет-магазин **Atlas Software** (asoft.kz) — сайт для продажи лицензионного программного обеспечения. Аналог softlist.biz, но с другой цветовой схемой и под казахстанский рынок.

### Страницы сайта

| Страница | URL | Описание |
|----------|-----|----------|
| Главная | `/` | Hero-баннер, категории, популярные товары, вендоры |
| Каталог | `/catalog` | Все товары с фильтрами и пагинацией |
| Категория | `/catalog/{slug}` | Товары конкретной категории |
| Товар | `/product/{slug}` | Страница товара с ценами |
| Корзина | `/cart` | Список выбранных товаров |
| Оформление | `/checkout` | Форма заказа |
| Войти | `/login` | Форма входа |
| Регистрация | `/register` | Форма регистрации |
| Личный кабинет | `/account` | История заказов |
| Избранное | `/favorites` | Список избранных |
| Вендоры | `/vendors` | Производители ПО |

---

## Технологический стек

### Backend: Laravel 11

**Laravel** — PHP-фреймворк, следующий паттерну MVC (Model-View-Controller). Мы используем его **только как API-сервер** (REST API). Он принимает HTTP-запросы, работает с базой данных и возвращает JSON.

```
Браузер (Vue.js) --HTTP-запрос--> Laravel API --SQL--> PostgreSQL
                 <--JSON-ответ---
```

**Почему Laravel:**
- Eloquent ORM — удобная работа с БД без написания SQL вручную
- Artisan CLI — генерация кода командами
- Sanctum — готовая аутентификация для SPA
- Migrations — версионирование схемы БД

### Frontend: Vue 3

**Vue.js** — JavaScript-фреймворк для построения интерфейсов. Мы строим **SPA (Single Page Application)** — страница загружается один раз, а переходы между разделами происходят без перезагрузки.

```
index.html --> app.js (Vue) --> компоненты --> рендер в браузере
                                 |
                           Vue Router (навигация)
                           Pinia (состояние)
                           Axios (HTTP к API)
```

**Почему Vue 3:**
- Composition API — современный способ писать логику компонентов
- Vite — молниеносная сборка (уже встроен в Laravel 11)
- Pinia — простой и типобезопасный state manager

### База данных: PostgreSQL

**PostgreSQL** — реляционная СУБД. Хранит все данные: товары, пользователей, заказы. Laravel общается с ней через Eloquent (ORM) — мы пишем PHP-код, а не SQL.

# Часть 2: Подготовка окружения {#часть-2}

## 2.1 Установка PHP 8.2+

Laravel 11 требует PHP версии 8.2 или выше.

### Windows (через XAMPP или отдельно)

Скачай [XAMPP](https://www.apachefriends.org/) — он включает PHP + Apache.
Либо скачай PHP отдельно с [windows.php.net](https://windows.php.net/download/).

После установки проверь версию:

```bash
php -v
# Должно показать: PHP 8.2.x или 8.3.x
```

### Расширения PHP, необходимые Laravel

Убедись, что в `php.ini` раскомментированы (убери `;` в начале строки):

```ini
extension=pdo_pgsql    ; для работы с PostgreSQL
extension=pgsql        ; драйвер PostgreSQL
extension=mbstring     ; работа со строками
extension=openssl      ; шифрование
extension=tokenizer    ; токенизатор PHP
extension=xml          ; работа с XML
extension=ctype        ; проверка типов символов
extension=curl         ; HTTP-запросы
extension=fileinfo     ; определение типа файлов
extension=zip          ; архивы (для Composer)
```

Файл `php.ini` обычно находится в `C:\xampp\php\php.ini`.

---

## 2.2 Установка Composer

**Composer** — менеджер зависимостей для PHP. Как npm для Node.js.

Скачай установщик с [getcomposer.org](https://getcomposer.org/download/).

После установки проверь:

```bash
composer --version
# Composer version 2.x.x
```

---

## 2.3 Установка Node.js

**Node.js** нужен для запуска Vite (сборщик frontend'а) и установки npm-пакетов (Vue, Tailwind и т.д.).

Скачай LTS-версию с [nodejs.org](https://nodejs.org/).

После установки:

```bash
node -v   # v20.x.x или выше
npm -v    # 10.x.x или выше
```

---

## 2.4 Установка PostgreSQL

Скачай с [postgresql.org](https://www.postgresql.org/download/windows/).

Во время установки:
- Запомни пароль для пользователя `postgres`
- Порт оставь `5432` (по умолчанию)
- Установи pgAdmin (графический интерфейс)

После установки открой **pgAdmin** или **psql** и создай базу данных:

```sql
-- Открой SQL-редактор в pgAdmin и выполни:
CREATE DATABASE asoft_kz;
CREATE USER asoft_user WITH PASSWORD 'secret_password';
GRANT ALL PRIVILEGES ON DATABASE asoft_kz TO asoft_user;
```

**Что это означает:**
- `CREATE DATABASE asoft_kz` — создаём базу данных с именем `asoft_kz`
- `CREATE USER` — создаём пользователя для Laravel (не используем суперпользователя `postgres` в целях безопасности)
- `GRANT ALL PRIVILEGES` — даём нашему пользователю полный доступ к этой базе

---

```bash
php -v          # PHP 8.2+

npm -v          # npm 10+
psql --version  # PostgreSQL 15+
```

# Часть 3: Создание Laravel-проекта {#часть-3}

## 3.1 Создание проекта

Перейди в папку, где хочешь создать проект (например `C:\PhpstormProjects\`), и выполни:

```bash
composer create-project laravel/laravel asoft.kz
```

**Что происходит:**
- `composer create-project` — команда Composer для создания проекта из шаблона
- `laravel/laravel` — официальный шаблон Laravel с Packagist (репозиторий PHP-пакетов)
- `asoft.kz` — имя папки проекта

После завершения перейди в папку проекта:

```bash
cd asoft.kz
```

## 3.3 Файл `.env` — главный конфигурационный файл

`.env` — это файл с переменными окружения. Laravel читает его при старте и использует значения в коде через функцию `env('КЛЮЧ')`. Это позволяет иметь разные настройки для разработки и продакшена.

Открой `.env` и приведи к такому виду:

```dotenv
# ============================================================
# Общие настройки приложения
# ============================================================

# Имя приложения — отображается в заголовках писем, логах
APP_NAME="Atlas Software"

# Режим работы:
#   local    — разработка (подробные ошибки в браузере)
#   production — продакшен (ошибки скрыты, кэш включён)
APP_ENV=local

# Секретный ключ шифрования. Генерируется командой:
# php artisan key:generate
# НИКОГДА не раскрывай этот ключ публично!
APP_KEY=

# Режим отладки. В production ВСЕГДА должен быть false
APP_DEBUG=true

# URL сайта. В разработке — localhost
APP_URL=http://localhost:8000

# ============================================================
# База данных
# ============================================================

# Тип СУБД. Мы используем PostgreSQL
DB_CONNECTION=pgsql

# Адрес сервера БД
DB_HOST=127.0.0.1

# Порт PostgreSQL (по умолчанию 5432)
DB_PORT=5432

# Имя базы данных, которую мы создали
DB_DATABASE=asoft_kz

# Пользователь БД
DB_USERNAME=asoft_user

# Пароль пользователя БД
DB_PASSWORD=secret_password

# ============================================================
# Кэш и очереди
# ============================================================

# Где хранить кэш (file — в файлах, redis — в Redis, database — в БД)
CACHE_STORE=file

# Где хранить очереди задач
QUEUE_CONNECTION=sync

# Где хранить сессии
SESSION_DRIVER=file

# Время жизни сессии в минутах (120 = 2 часа)
SESSION_LIFETIME=120

# ============================================================
# Почта
# ============================================================

# Драйвер почты. log — просто пишет письма в лог-файл (для разработки)
MAIL_MAILER=log
MAIL_FROM_ADDRESS="noreply@asoft.kz"
MAIL_FROM_NAME="Atlas Software"

# ============================================================
# Хранилище файлов (изображения товаров и т.д.)
# ============================================================

# local — хранить в папке storage/app
FILESYSTEM_DISK=local
```

После заполнения `.env` сгенерируй секретный ключ:

```bash
php artisan key:generate
```

**Что делает эта команда:** генерирует случайный 32-байтовый ключ и записывает его в `APP_KEY`. Он используется для шифрования сессий, кук и других данных.

## 3.4 Первый запуск

```bash
php artisan serve
```

Это запускает встроенный PHP-сервер. Открой браузер: `http://localhost:8000`.

Ты увидишь стартовую страницу Laravel. Значит всё работает.

**`php artisan`** — это CLI-инструмент Laravel. `artisan` — файл в корне проекта. Через него мы делаем почти всё: создаём файлы, запускаем миграции, очищаем кэш.

# Часть 4: База данных PostgreSQL {#часть-4}

## 4.1 Архитектура базы данных

Нарисуем схему (ER-диаграмма):

```
users
  └─── orders (один пользователь → много заказов)
         └─── order_items (один заказ → много позиций)
                └─── products

products
  ├─── categories (много товаров → одна категория)
  ├─── vendors    (много товаров → один вендор)
  ├─── product_images (один товар → много изображений)
  └─── product_licenses (один товар → много типов лицензий с ценами)

cart_items
  ├─── users     (чья корзина)
  └─── products  (какой товар)

favorites
  ├─── users
  └─── products
```

---

## 4.2 Таблицы и их назначение

| Таблица | Назначение |
|---------|-----------|
| `users` | Покупатели и администраторы |
| `categories` | Иерархический каталог (САПР → AutoCAD) |
| `vendors` | Производители ПО (Microsoft, Kaspersky...) |
| `products` | Товары (программы) |
| `product_images` | Изображения товара |
| `product_licenses` | Типы лицензий с ценами (1 ПК, 5 ПК...) |
| `cart_items` | Корзина (товары, добавленные но не купленные) |
| `favorites` | Избранные товары |
| `orders` | Оформленные заказы |
| `order_items` | Позиции в заказе |
