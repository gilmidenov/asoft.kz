# Руководство по работе с Административной панелью

## Доступ к панели

Административная панель доступна по адресу **http://localhost:8000/admin**.

### Требования для доступа

Чтобы войти в панель, пользователь должен:
1. Быть зарегистрирован в системе
2. Иметь роль `admin` в поле `role` таблицы `users`

### Создание администратора

Установить роль администратора можно через Artisan Tinker:

```bash
docker exec -it asoft_app php artisan tinker
```

```php
// Установить admin для существующего пользователя
User::where('email', 'your@email.com')->update(['role' => 'admin']);

// Или создать нового администратора
User::create([
    'name'     => 'Admin',
    'email'    => 'admin@asoft.kz',
    'password' => bcrypt('your_password'),
    'role'     => 'admin',
]);
```

## Разделы панели

### Дашборд (`/admin`)

Стартовая страница с ключевыми показателями:
- Общее количество товаров, категорий, вендоров, заказов
- Таблица последних 5 заказов с быстрым переходом в раздел Заказы

---

### Товары (`/admin/products`)

Управление каталогом программного обеспечения.

#### Создание товара

Нажмите «+ Добавить товар». В форме заполните:

| Поле | Описание | Обязательно |
|------|----------|-------------|
| Название | Полное коммерческое название продукта | Да |
| Категория | Выберите из списка существующих | Нет |
| Вендор | Производитель/правообладатель | Нет |
| Краткое описание | 1-2 предложения для карточки товара | Нет |
| Описание | Полный текст для страницы товара | Нет |
| Версия | Например: `2024`, `11 Pro` | Нет |
| Язык | Например: `Русский`, `Multilingual` | Нет |
| Доставка | `Ключ` / `Загрузка` / `Коробка` | Нет |
| Статус | `Активен` / `Неактивен` / `Нет в наличии` | Да |
| Хит продаж | Отображается в секции «Хиты» на главной | Нет |
| Новинка | Помечается тегом «Новинка» | Нет |
| Акция | Отображается в разделе «Акции» | Нет |

> **Slug** (URL-адрес товара) генерируется автоматически из названия.

#### Редактирование товара

Нажмите «Изменить» напротив нужного товара. Доступны те же поля, что и при создании.

#### Удаление товара

Нажмите «Удалить» и подтвердите действие. **Удаление необратимо.**

### Категории (`/admin/categories`)

Иерархическая структура каталога (родительские категории и подкатегории).

#### Создание категории

| Поле | Описание |
|------|----------|
| Название | Отображаемое имя категории |
| Родительская категория | Для создания подкатегории выберите родителя |
| Описание | Краткое описание категории |
| Изображение (URL) | Ссылка на изображение для страницы категории |
| Порядок сортировки | Число — чем меньше, тем выше в списке |
| Активна | Снимите флажок, чтобы скрыть категорию с сайта |

> Slug генерируется автоматически. Подкатегории отображаются в таблице с отступом (└).

---

### Вендоры (`/admin/vendors`)

Управление производителями и правообладателями ПО.

| Поле | Описание |
|------|----------|
| Название | Официальное название компании (Microsoft, Adobe...) |
| Описание | Краткая информация о вендоре |
| Логотип (URL) | Ссылка на изображение логотипа |
| Сайт | Официальный сайт (с https://) |
| Активен | Скрыть/показать вендора на сайте |

---

### Заказы (`/admin/orders`)

Просмотр и управление всеми заказами покупателей.

#### Фильтрация по статусу

Используйте выпадающий список «Все статусы» для фильтрации заказов.

#### Статусы заказов

| Статус | Описание |
|--------|----------|
| `Ожидает оплаты` | Заказ создан, оплата не получена |
| `Оплачен` | Платёж подтверждён |
| `Обрабатывается` | Готовится к отправке |
| `Выполнен` | Ключи/ссылки переданы клиенту |
| `Отменён` | Заказ отменён |
| `Возврат` | Оформлен возврат средств |

#### Изменение статуса

В колонке «Изменить статус» выберите новый статус из выпадающего списка. Изменение применяется **немедленно** без дополнительного подтверждения.

## API Endpoints для администраторов

Все admin-маршруты требуют заголовок `Authorization: Bearer {token}` и роль `admin`.

```
GET    /api/admin/orders              — список всех заказов (с фильтром ?status=)
PATCH  /api/admin/orders/{id}/status  — изменить статус заказа

POST   /api/admin/categories          — создать категорию
PUT    /api/admin/categories/{id}     — обновить категорию
DELETE /api/admin/categories/{id}     — удалить категорию

POST   /api/admin/vendors             — создать вендора
PUT    /api/admin/vendors/{id}        — обновить вендора
DELETE /api/admin/vendors/{id}        — удалить вендора

POST   /api/admin/products            — создать товар
PUT    /api/admin/products/{id}       — обновить товар
DELETE /api/admin/products/{id}       — удалить товар
```

---

## Безопасность

- Все admin-маршруты защищены middleware `auth:sanctum` + `role:admin`
- Пользователь с ролью `customer` получит HTTP 403 при любом обращении к admin API
- Неавторизованный пользователь получит HTTP 401
